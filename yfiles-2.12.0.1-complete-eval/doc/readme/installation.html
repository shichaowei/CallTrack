yFiles for Java in your Project

Installing the yFiles Distribution

yFiles Library

In order to use the yFiles library you need to include the y.jar file in your Java classpath. It can be found in the lib/ directory.

You can verify that the installation was correct by running the demo programs that are part of the yFiles distribution.

Note that the resource files located in the demo subfolders or the tutorial subfolder need to be in your classpath in order for the demos to properly function. IDEA users should add at least the resource patterns ?*.png;?*.jpg;?*.gif;?*.gml;?*.ygf;?*.graphml;?*.graphmlz;?*.xml;?*.xsl;?*.csv;?*.ged;?*.owl to their compiler settings.

Documentation

To get quick access to the API documentation in your IDE, you'll want to add the API documentation to the y.jar file:

IDEA: Add the y.jar file in the Library or Global Library tab of the project settings dialog. Then press the "Add Documentation" button and choose the doc/api/ folder.

Eclipse: In the Libraries tab of the "Java Build Path" for your project, select the y.jar file, expand the (+) sign and set the Javadoc path to the doc/api/ folder.

NetBeans: From the Tools menu, choose "Libraries" to open the Library Manager dialog. Add the y.jar file in the Classpath tab, then attach the doc/api/ folder in the Javadoc tab.
