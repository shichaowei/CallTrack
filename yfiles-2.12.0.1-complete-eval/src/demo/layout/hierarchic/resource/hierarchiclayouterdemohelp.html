Description

This demo shows how to use the capabilities of the HierarchicLayouter on port constraints, grouped nodes and edge label positioning.

For an edge, port constraints specify on which side the edge connects to an incident node. There are independent port constraints for both, the source and the target node. Thus, it is possible to affect only one side, either the source or the target node.
If a port constraint is marked as strong, the exact position of the port will be retained.
Edges incident to a common node can be unified at this common node by being assigned to the same (edge) group.

For nodes, node groups result in nodes being placed horizontally next to each other if the nodes are on the same layer, i.e. on a shared vertical position.

Usage

Parse Ports
Source and target port constraints will be set for each edge in the graph. This option is also known as create port constraints from sketch.
Label Edges
Displays edge labels that show the current source and target port constraints.
Additionally, the edge labels demonstrate how to specify the preferred label placement automatic label placement.
Click Layout while edge labels are shown to re-arrange the graph with port constraints corresponding to the shown labels.
Source-/Target Port
Select one edge via left-click to see or set its source and target port constraints.
Re-arrange the graph by clicking Layout to apply any changed constraints.
Edge Groups
Assigns the selected edges to source and/or target edge group. All edges incident to a node share the same edge path when assigned to a common edge group.
Re-arrange the graph by clicking Layout the see any effects.
Node Groups
Assigns selected nodes to a common group. Nodes of a common group and a common layer will be placed horizontally next to each other.
Re-arrange the graph by clicking Layout the see any effects.
Layout
Arranges the graph with the Hierarchical Layout algorithm respecting given constraint and group settings.
Settings
Further options for the Hierarchic Layout algorithm.
