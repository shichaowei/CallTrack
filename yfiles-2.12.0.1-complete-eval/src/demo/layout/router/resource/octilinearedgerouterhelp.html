Description

This demo shows yFiles' octilinear edge routing capabilities. An edge routing algorithm routes edges without changing the current node positions. While an orthogonal edge routing algorithm only produces horizontal and vertical edge segments, this router also allows octilinear edge segments, i.e., it produces edge routes where the slope of each segment is a multiple of 45 degrees. Besides the basic octilinear edge routing capabilities, this class also demonstrates the edge grouping feature. For clarity, edges of the same edge group are displayed in a common color.

Usage

Create edge groups either by selecting multiple edges and choosing item "Group Selected Edges" in the context menu or by selecting a node and choosing item "Group In-Edges" or "Group Out-Edges" in the context menu. Note that edges can be either grouped at their source or at their target (not both at the same time). Press "Route" to route the edges.

Available Options

Automatic Routing
If this option is enabled, the edge routing is automatically triggered when creating new edges, changing edges or moving/resizing nodes.
Octilinear Routing
If this option is enabled, the algorithm also produces octilinear (non-orthogonal) edge segments.
Preferred Octilinear Segment Length
Specifies the preferred maximum length of non-orthogonal edge segments.
Ignore Edge Groups
If this option is enabled, the algorithm ignores the specified edge groups.
Minimum Edge Distance
Specifies the minimum distance between two edges.
Minimum Node Distance
Specifies the minimum distance between an edge and a node.
